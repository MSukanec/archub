Quiero que implementes una funcionalidad swipe para mobile como en WhatsApp o Mail de iOS.

---

🎯 OBJETIVO

- En mobile, todas las cards dentro de `src/components/cards/` deben permitir deslizar hacia la izquierda para mostrar acciones (ej. Editar, Eliminar, Favorito).
- Esta funcionalidad debe estar encapsulada en un nuevo componente llamado:

src/components/layout/mobile/SwipeableCard.tsx

yaml
Copiar
Editar

---

### 🛠 FUNCIONALIDAD DE `SwipeableCard.tsx`

1. Este componente debe aceptar como props:

```tsx
{
  children: ReactNode; // el contenido principal de la card
  actions?: {
    label: string;
    icon: ReactNode;
    variant?: "destructive" | "default";
    onClick: () => void;
  }[];
}
Por defecto, si no se pasa ninguna acción, debe mostrar:

⭐ Marcar como favorito

✏️ Editar

🗑️ Eliminar

Cada uno debe funcionar igual que en desktop (marcar favorito, abrir modal de edición, o abrir modal de confirmación para borrar).

Debe tener comportamiento táctil swipe izquierdo, compatible con react-swipeable, framer-motion o tu método preferido.

📱 REQUISITOS VISUALES
Que las acciones se revelen al deslizar la card hacia la izquierda

Que el contenido de la card quede encima, como en WhatsApp

Que los botones aparezcan en fondo gris claro o destructivo (rojo) según el variant

🧩 USO EN CUALQUIER CARD
En todas las cards de src/components/cards/, deben poder reemplazar el return por algo como:

tsx
Copiar
Editar
<SwipeableCard
  actions={[
    {
      label: "Editar",
      icon: <EditIcon />,
      onClick: () => openEditModal(data),
    },
    {
      label: "Eliminar",
      icon: <TrashIcon />,
      variant: "destructive",
      onClick: () => openDeleteModal(data),
    },
  ]}
>
  {/* contenido de la card */}
</SwipeableCard>
📌 El objetivo es que cualquier card pueda usar SwipeableCard y tener swipe-to-action en mobile. En desktop este comportamiento NO debe mostrarse.

Podés usar useMediaQuery() o clases md:hidden para limitar esto a mobile si hace falta.
