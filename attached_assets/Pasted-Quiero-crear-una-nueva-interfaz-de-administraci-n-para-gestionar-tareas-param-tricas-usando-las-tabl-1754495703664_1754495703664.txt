Quiero crear una nueva interfaz de administración para gestionar tareas paramétricas usando las tablas `task_templates` y `task_template_parameters`.

---

🎯 OBJETIVO

Diseñar una interfaz clara, simple y escalable para administrar plantillas de tareas paramétricas.

---

🧱 TABLAS

1. `task_templates`
   - id
   - name
   - slug
   - description
   - unit_id
   - name_expression (ej: 'Muro {tipo_muro} con {tipo_ladrillo}')
   - is_active
   - created_by
   - created_at

2. `task_template_parameters`
   - id
   - template_id
   - parameter_id
   - order_index
   - is_required
   - condition_json
   - created_at

3. `task_parameters` (ya existente)
   - id
   - name
   - slug
   - type, etc.

---

🧩 COMPONENTES A CREAR

1. **Página de listado de plantillas**
   - Tabla con columnas: `name`, `slug`, `unidad`, `cantidad de parámetros`, `acciones`
   - Botón para crear nueva plantilla (abre modal o panel)
   - Acciones: editar, eliminar

2. **Página de edición de plantilla (detalle)**
   - Campo: `name`
   - Campo: `slug`
   - Selector: `unit_id` (dropdown)
   - Textarea: `name_expression`
   - Campo: `description`
   - Toggle: `is_active`
   - Sección: `Parámetros`

3. **Gestión de parámetros dentro de la plantilla**
   - Tabla con columnas: `Parámetro`, `Orden`, `Requerido (checkbox)`, `Condición (JSON editable)`, `Acciones`
   - Botón “Agregar parámetro” abre modal con `task_parameters` existentes
   - Permitir drag & drop para reordenar `order_index`
   - Validaciones: no duplicar parámetros

4. **Vista previa de nombre generado**
   - Con `name_expression` y valores de ejemplo
   - Ej: `Muro doble hueco 18 con cemento con hidrófugo`

---

🧑‍💻 EXTRAS (opcionales)

- Permitir duplicar una plantilla existente
- Permitir probar combinaciones desde esta interfaz
- Agregar validación visual si `name_expression` usa slugs que no están en la plantilla

---

🌙 Estilo: dark mode, limpio, coherente con el resto de Archub
🧠 UX: clara, legible, intuitiva

Esta interfaz reemplaza la necesidad de crear tareas paramétricas desde estructuras complejas como nodos o árboles. Todo se configura desde estas plantillas y sus parámetros.
